## Database Schema

Materials are stored in the `materiales` table:

```sql
CREATE TABLE materiales (
  id               uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  codigo           text UNIQUE,
  nombre           text NOT NULL,
  categoria        categoria_enum NOT NULL,
  unidad_base      text NOT NULL DEFAULT 'kgs',
  peso_aproximado  numeric,
  envase           text,
  orden            integer NOT NULL
);

CREATE TYPE categoria_enum AS ENUM (
  'materia_prima', 'esencia', 'varios', 'envase_vacio', 'color'
);
```

### Querying Materials

**Get all materials by category:**
```sql
SELECT * FROM materiales 
WHERE categoria = 'esencia' 
ORDER BY orden;
```

**Search by name:**
```sql
SELECT * FROM materiales 
WHERE nombre ILIKE '%lavanda%';
```

**Get materials with specific packaging:**
```sql
SELECT * FROM materiales 
WHERE envase = '200lts' 
ORDER BY categoria, orden;
```

## Usage Notes

### Ordering

1. **Kilograms vs Pieces:** Check `unidad_base` field:
   - `kgs` or `Lt`: Order by weight
   - `Pzas` or `Paquete`: Order by quantity

2. **Container Sizes:** The `peso_aproximado` field indicates container size, not unit price

3. **Legacy Codes:** Some materials have `codigo` values (legacy system), but most use only `nombre`

### Display Order

Materials are sorted by the `orden` field within each category:

```typescript
const materials = await supabase
  .from('materiales')
  .select('*')
  .eq('categoria', 'esencia')
  .order('orden', { ascending: true });
```

### Material Availability

All materials in this catalog are available for ordering. Future versions may include an `activo` field to mark discontinued items.
